LangGraph RAG Pipeline

6 nodes · 4 edges · 1 branch · 1 loop

Node (a step/function)
Edge (always goes here)
Conditional (branch)
Conditional (loop/retry)

NODES 6 steps (functions)

# Each node is a Python function
# that takes state → returns updates

graph.add_node("rewrite_query", rewrite_query)
graph.add_node("retrieve", retrieve)
graph.add_node("grade_documents", grade_documents)
graph.add_node("websearch_fallback", websearch_fallback)
graph.add_node("generate", generate)
graph.add_node("check_hallucination", check_hallucination)

EDGES 4 fixed connections

# "After A, always go to B"

graph.add_edge(START, "rewrite_query")
graph.add_edge("rewrite_query", "retrieve")
graph.add_edge("retrieve", "grade_documents")
graph.add_edge("websearch_fallback", "generate")
graph.add_edge("generate", "check_hallucination")

BRANCH Conditional routing

# "After grading, pick ONE path"

graph.add_conditional_edges(
    "grade_documents",
    route_after_grading,  # decides which way
    {
      "generate": "generate",
      "websearch_fallback": "websearch_fallback",
    }
)

LOOP Retry cycle

# "If hallucinated, go BACK to generate"
# This is a CYCLE — the key LangGraph feature

graph.add_conditional_edges(
    "check_hallucination",
    route_after_check,  # decides: done or retry
    {
      "accept": END,           # ✅ done!
      "retry": "generate",    # ⚠️ try again
    }
)

6 Nodes = the steps (Python functions)  ·  4 Edges = fixed arrows (always go here)  ·  1 Branch = pick a path based on a condition  ·  1 Loop = go back and retry (the key LangGraph feature)
